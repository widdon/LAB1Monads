# Лабораторная работа №1

**Тема:** Монады в функциональном программировании  
**Вариант 6:** Парковка

# Цель работы

Цель лабораторной работы — изучить и применить монады в языке программирования Scala.

В ходе работы необходимо:

- реализовать собственные монады;
- научиться разделять чистые вычисления и эффекты;
- моделировать состояние системы в функциональном стиле;
- организовать взаимодействие с пользователем через безопасные эффекты;
- построить приложение без использования императивного подхода. [web:2][web:5]

---

# Постановка задачи

Необходимо реализовать систему управления парковкой.

Программа должна:

- принимать автомобили на парковку;
- выпускать автомобили;
- рассчитывать стоимость парковки;
- учитывать штраф за потерянный билет;
- хранить состояние парковки;
- вести лог действий;
- взаимодействовать с пользователем через консоль.

# Реализация монад

## Monad

Реализован trait `Monad[M[_]]`, содержащий основные операции:

- `pure`;
- `flatMap`;
- `map`.

Назначение:

- `pure` — поднимает значение в контекст монады;
- `flatMap` — связывает вычисления;
- `map` — преобразует значение внутри контекста.

## Reader

Монада Reader используется для передачи конфигурации парковки.

Хранимая конфигурация:

- число мест;
- стоимость часа;
- штраф;
- правило округления времени.

Использование:

- расчёт стоимости;
- проверка возможности въезда;
- получение штрафа.

## Writer

Монада Writer используется для накопления логов.

Логируются:

- въезд автомобиля;
- назначение места;
- расчёт стоимости;
- выезд;
- штрафы;
- ошибки.

В качестве хранилища логов используется `Vector[String]`.

## State

Монада State хранит текущее состояние парковки.

Состояние включает:

- занятые места;
- карту автомобилей;
- текущее время;
- выручку.

## IO

Монада IO используется для:

- чтения пользовательского ввода;
- вывода информации в консоль;
- отделения описания эффекта от его выполнения.

Все эффекты выполняются только через `unsafeRun()`.

# Реализация предметной области

## ParkingConfig

Класс содержит конфигурацию парковки:

- `capacity`;
- `hourlyRate`;
- `lostTicketFine`;
- `roundUpToHour`.

## ParkingState

Хранит текущее состояние системы:

- `occupiedSpots`;
- `entryHours`;
- `spotByCar`;
- `currentHour`;
- `profit`.

## ParkingLogic

Основная бизнес-логика приложения.

Реализованы функции:

| Функция | Назначение |
| --- | --- |
| `enterCar` | Въезд машины |
| `exitCar` | Выезд машины |
| `nextHour` | Перевод времени |
| `reportTicket` | Потерянный билет |

---

## Рефакторинг логики

Изначально функции содержали:

- большие `if / else`;
- глубокую вложенность;
- дублирование кода.

После рефакторинга были выделены helper-функции:

- `enterError`;
- `exitError`;
- `successfulEnter`;
- `successfulExit`;
- `successfulLostTicket`.

Это позволило:

- уменьшить nesting;
- убрать дублирование;
- сделать код более декларативным;
- повысить читаемость.

# Реализация пользовательского интерфейса

## Архитектура меню

Меню реализовано в виде дерева объектов.

Используются:

- `MenuOption`;
- `MenuLeaf`;
- `MenuTreeNode`;
- `UserInteraction`.

## MenuOption

Абстрактный пункт меню:

```scala
trait MenuOption
```

Содержит:

- название;
- действие.

## MenuLeaf

Лист дерева меню.

Представляет конкретное действие:

- въезд;
- выезд;
- штраф;
- перевод времени.

## MenuTreeNode

Узел дерева меню.

Отвечает за:

- отображение меню;
- обработку выбора;
- запуск пользовательского цикла.

## UserInteraction

Содержит:

- цикл взаимодействия;
- чтение команд;
- рекурсивное выполнение меню.

# Особенности функционального подхода

В работе полностью исключён императивный стиль.

Не используются:

- `var`;
- mutable-коллекции;
- циклы `while`;
- глобальные переменные.

Используются:

- immutable-состояние;
- рекурсия;
- монады;
- композиция функций;
- pure-функции.
